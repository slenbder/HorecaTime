# HorecaTime — Telegram-бот учёта рабочего времени

## Суть проекта
Перенос Telegram-бота с Google Apps Script на Python (aiogram 3.x).
Apps Script версия — рабочий референс в продакшне. Файлы скриптов доступны в ассетах проекта.

---

## Стек
- Python 3.11+, aiogram 3.13.1
- gspread 6.1.2, oauth2client 4.1.3
- APScheduler 3.10.4
- aiosqlite 0.20.0
- SQLite — только FSM + кеш ролей, НЕ основная БД
- Google Sheets — единственный источник данных (single source of truth)

## Логирование — ОБЯЗАТЕЛЬНЫЕ ПРАВИЛА

Три лог-файла, настроены в `app/logging_config.py`:
- `logs/app.log` — общий поток: старт, действия пользователей, успешные операции
- `logs/errors.log` — только ошибки и исключения
- `logs/googleapi.log` — все обращения к Google Sheets (запросы, ответы, реконнекты)

**Всегда логировать:**
- Старт и остановку бота
- Каждый вызов Google Sheets API (в googleapi.log)
- Реконнекты (`_reconnect()`)
- Approve/reject пользователя
- Добавление пользователя в месячный лист
- Критические ошибки → лог + Telegram-алерт разработчику

**Никогда не использовать** `print()` — только `logging`.

## Google Sheets — структура

**Техлист** (колонки):
- A(1): Telegram ID
- B(2): ник
- C(3): TG-имя
- D(4): последнее сообщение
- E(5): registered_at
- F(6): last_seen_at
- G(7): message_id
- H(8): ФИО от пользователя
- I(9): этап взаимодействия (0/1/2/3)
- J(10): «ДА/НЕТ» — в штате
- K(11): отдел (Зал/Бар/Кухня)
- L(12): позиция

**Месячный лист** («Часы», «Январь 2025» и т.д.):
- A: ФИО, B: Telegram ID, C: позиция
- Строка 3: даты месяца
- Данные: D5:R60 и T5:AI60
- Колонка 38: часовая ставка
- Ключевые ячейки: C2 (месяц), T2 (год)

## Бизнес-логика — ключевые правила

**Расчёт часов:**
- `H = endTime - startTime`, округление до 0.5 (`roundToHalf`)
- Пересечение полуночи обрабатывается: `H = 24 - start + end`
- Выходные: пятница, суббота, воскресенье — ставка выходного дня

**Форматы ввода даты:** `1.1` / `01.01` / `1.01.25` / `01.01.25`
**Форматы ввода времени:** `1000-1800` / `10-18` / `10.00-18.00`
**Спецзначения:** `X` = рабочий день (без часов), `О/o/0` = нулевая смена

**Добавление в месячный лист:**
1. Найти секцию по позиции (`POSITION_TO_SECTION`)
2. Fallback: конец блока отдела (`DEPARTMENT_TO_HEADER`)
3. Fallback: конец листа
4. Защита от дублей — проверка по Telegram ID в колонке B

## Механика H/AH — бизнес-логика по ролям

## Механика H/AH по позициям

Формат ячейки в месячном листе: H/AH (например 8/3). Если AH=0 → пишем H/0.
Нулевая смена → пустая ячейка. Спецзначение X — не используется.

| Позиция       | H  | AH                                          | Повышенная ставка (пт/сб/вс) | Ввод нескольких смен |
|---------------|----|---------------------------------------------|------------------------------|----------------------|
| Кухня (все)   | ✅ | ❌                                          | ❌                           | Одним сообщением     |
| Хостесс       | ✅ | ❌                                          | ❌                           | Одним сообщением     |
| Менеджер      | ✅ | ❌                                          | ❌ (уточнить позже)          | Пошагово             |
| Официант      | ✅ | ✅ карты/чеки (0.5ч/фото, апрув admin_hall) | ❌                           | Пошагово             |
| Раннер        | ✅ | ✅ доп.часы через бота (всегда базовая)     | ✅ только H                  | Пошагово             |
| Бармен/Барбэк | ✅ | ✅ тусовочные (проверка нахлёста)           | ❌                           | Пошагово             |

### Детали по позициям

**Раннер:**
- H = часы смены, округление до 0.5
- Повышенная ставка на H в пт/сб/вс (определяется по дате на Python)
- AH = доп.часы, вводит пользователь через бота после основной смены
- После ввода AH бот запрашивает комментарий, который включается в уведомление админу
- AH всегда оплачиваются по базовой ставке, даже в выходные
- Запись: H/AH (например 8/5). В уведомлении админам: H отдельно, AH отдельно с комментарием

**Официант:**
- H = часы смены, одна базовая ставка (без повышенной в выходные)
- AH = доп.часы за карты лояльности и чеки >8000р
- Механика: пользователь прикрепляет N фото → каждое фото = 0.5ч AH
- Фото пересылаются admin_hall для апрува/отклонения каждого отдельно
- После апрува admin_hall → +0.5ч AH записывается в таблицу
- Запись: H/AH

**Бармен/Барбэк:**
- H = обычные часы смены (базовая ставка)
- AH = тусовочные часы (повышенная/тусовочная ставка)
- День недели не влияет на ставки
- Проверка нахлёста: H и AH не могут пересекаться по времени
- При нахлёсте → алерт пользователю с просьбой исправить диапазоны
- Запись: H/AH (например 3/9). Если тусовки не было → H/0

**Кухня / Хостесс:**
- Только H, одна ставка, без повышенных и без AH
- Несколько смен можно внести одним сообщением (несколько строк)

**Менеджер:**
- Пока как Кухня/Хостесс (только H), детали уточнятся позже

## Команды по ролям

**user (все позиции включая Менеджера):**
- Внести смену(ы)
- Часы за текущий месяц (1-я половина / 2-я половина) — только часы, без рублей
- Часы за прошлый месяц — только часы, без рублей
- Заработок — отдельная команда
- График — без редактирования (этап PDF)

**admin_hall** (подчинение: Менеджер, Официант, Раннер, Хостесс):
- Все команды user + свои смены и свои часы
- График — со ссылкой на редактирование
- Ставки своего отдела (базовая + повышенная)
- Изменить ставку своего отдела
- Сообщение сотрудникам своего отдела
- Апрув фото карт/чеков от официантов

**admin_bar** (Бармен, Барбэк):
- Аналогично admin_hall для своего отдела (без апрува фото)

**admin_kitchen** (Горячий, Холодный, Кондитерский, Заготовочный, Коренной, МОП — без Су-шефа):
- Аналогично admin_hall для своего отдела

**superadmin** (только управляющий, смены не вносит):
- График — без редактирования
- Ставки всех отделов включая Су-шефа (базовая + повышенная)
- Изменить ставки всех отделов включая Су-шефа
- Сообщение: выбрать отдел ИЛИ всем сразу
- Переход на новый месяц + копировать лист Часы

**developer:** всё как superadmin + алерты об ошибках

**Найм/увольнение:** обсуждается после реализации всего остального функционала
