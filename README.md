## О проекте

**HorecaTime** — корпоративный Telegram-бот для ресторана, который заменяет ручной учёт смен в таблицах. Сотрудники вносят свои смены прямо из Telegram, администраторы мгновенно получают уведомления, а Google Sheets автоматически считает часы и заработок.

Бот написан на Python (aiogram 3) и является портом рабочей системы на Google Apps Script. Google Sheets остаются единственным источником данных — ни одна цифра не теряется.

## Функционал

### 👤 Для сотрудников
- `/shift` — внести смену. Принимает любой формат: `1.3 10-18`, `01.03 10:00-18:00`, `1.03.26 10.00-18.00` и даже тире с пробелами
- `/hours_first` / `/hours_second` — часы и заработок за первую/вторую половину месяца
- `/hours_last` — итог за прошлый месяц с заработком по зафиксированной ставке
- `/schedule` — PDF-график своего отдела прямо в чат
- `/contact_dev` — написать разработчику

### 🔧 Особенности по позициям
| Позиция | Механика |
|---|---|
| Кухня / Хостесс / МОП | Только H, одна ставка, можно внести несколько смен одним сообщением |
| Раннер | H + доп. часы с комментарием, повышенная ставка в пт/сб/вс |
| Бармен / Барбэк | H + тусовочные часы, проверка нахлёста диапазонов |
| Официант | H + фото чеков/карт → апрув admin_hall → доп. часы |

### 🔑 Для администраторов отдела
- Все команды сотрудников + свои смены и часы
- `/rates` / `/set_rate` — ставки своего отдела
- `/message_dept` — рассылка сотрудникам отдела
- `/schedule` — PDF-график своего блока (Кухня/Бар/Зал/МОП)
- Апрув фото карт и чеков от официантов (только admin_hall)

### 👑 Для суперадмина
- `/rates_all` / `/set_rate_all` — ставки всех отделов включая Шеф/Су-шеф
- `/message_all` — рассылка всем сотрудникам сразу
- `/message_dept` — рассылка конкретному отделу
- `/dismiss` — увольнение сотрудника в 3 клика (с подтверждением)
- `/switch_month` — переключение на новый месяц вручную
- `/schedule` — PDF всего месячного листа целиком

### 🤖 Автоматика
- **1-е число в 12:00 МСК** — уведомление всем: «через 6 часов переключится месяц»
- **1-е число в 18:00 МСК** — автоматическое переключение месяца:
  - Копирует текущий лист, очищает смены, переносит активных сотрудников
  - Уволенные (красные строки) не переносятся
  - Фиксирует снимок ставок в архив (для корректного расчёта зарплаты за прошлый месяц)

## Иерархия ролей

```
Developer
    └── Superadmin (управляющий)
            ├── Admin Hall (Зал + МОП)
            ├── Admin Bar (Бар)
            └── Admin Kitchen (Кухня + Доп.)
                        └── User (сотрудники)
```

**Регистрация:**
- Суперадмин — только через `.env` (разработчик добавляет вручную)
- Администратор отдела — через бота, апрув суперадмином + указывает Gmail для доступа к таблице
- Сотрудник — через бота, апрув администратором или суперадмином

## Быстрый старт

### 1. Клонируй репозиторий
```bash
git clone https://github.com/slenbder/HorecaTime.git
cd HorecaTime
```

### 2. Создай виртуальное окружение
```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Настрой конфигурацию
```bash
cp .env.example .env
# Заполни .env (см. раздел Конфигурация)
```

### 4. Запусти бота
```bash
python main.py
```

## Конфигурация

Создай файл `.env` по образцу:

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | Токен Telegram бота (от @BotFather) |
| `GOOGLE_CREDENTIALS_PATH` | Путь к JSON сервисного аккаунта Google |
| `SPREADSHEET_ID` | ID Google таблицы (из URL) |
| `SUPERADMIN_IDS` | Telegram ID суперадминов через запятую |
| `DEVELOPER_ID` | Telegram ID разработчика |
| `ADMIN_HALL_IDS` | Telegram ID администраторов Зала |
| `ADMIN_BAR_IDS` | Telegram ID администраторов Бара |
| `ADMIN_KITCHEN_IDS` | Telegram ID администраторов Кухни |
| `SHEET_URL` | Ссылка на таблицу (отправляется сотрудникам при апруве) |

### Google Sheets — настройка доступа

1. Создай сервисный аккаунт в Google Cloud Console
2. Скачай JSON ключ → укажи путь в `GOOGLE_CREDENTIALS_PATH`
3. Дай сервисному аккаунту доступ **Редактор** к таблице
4. Настрой защиту Техлиста (только сервисный аккаунт + суперадмины могут редактировать)
5. Скрой Техлист от обычных пользователей (ПКМ на вкладке → Скрыть лист)

## Фишки

- **Умный пример ввода** — `/shift` показывает сегодняшнюю дату и последний прошедший получас
- **Любой формат тире** — `10:00-18:00`, `10:00 - 18:00`, `10:00–18:00`, `10:00—18:00` — всё работает
- **Смена через полночь** — `22:00-03:00` считается правильно (5 часов, не минус 19)
- **Кликабельные ники** — все уведомления содержат `@ник` со ссылкой на профиль
- **Красные строки** — уволенные сотрудники остаются в архиве с розовым фоном, в новый месяц не переносятся
- **Двойная проверка при переходе месяца** — цвет строки + наличие в Техлисте
- **История ставок** — при переключении месяца ставки фиксируются, `/hours_last` считает зарплату по ставке того месяца
- **Rotating logs** — логи автоматически ротируются (5MB × 7 файлов)
- **Авторазмер ячеек** — при добавлении сотрудника таблица сама подстраивает ширину колонок
