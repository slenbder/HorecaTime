# 📋 Инструкция — Администраторы

> Это руководство для **администраторов отделов**: Зала, Бара и Кухни.  
> Описывает расширенные функции, доступные после повышения вас суперадмином.

---

## Что умеет администратор

Администратор — это сотрудник своего отдела с расширенными правами. Он:
- одобряет или отклоняет заявки новых сотрудников своего отдела
- одобряет доп. часы, карты лояльности и наполняемость чеков (для Зала)
- видит и устанавливает ставки сотрудников своего отдела
- делает рассылки по отделу
- при этом **сохраняет все обычные функции сотрудника** — может вносить собственные смены и смотреть свои часы и расчёт заплаты

## Доступные команды

| Команда | Что делает |
|---|---|
| `/shift` | Внести собственную смену |
| `/hours_first` | Свои часы и заработок за 1–15 |
| `/hours_second` | Свои часы и заработок за 16–конец + итог |
| `/hours_last` | Свои часы и заработок за прошлый месяц |
| `/rates` | Ставки сотрудников своего отдела |
| `/set_rate` | Изменить ставку сотруднику |
| `/message_dept` | Рассылка по своему отделу |
| `/schedule` | График работы (только свой блок) |
| `/sheet` | Ссылка на таблицу с графиком |
| `/contact_dev` | Написать разработчику |

## Заявки новых сотрудников

Когда новый сотрудник проходит регистрацию в боте, администратор отдела получает уведомление:

```
📝 Новая заявка на доступ к боту:

👤 ФИО: Иванов Иван
🏢 Отдел: Зал
💼 Позиция: Официант
🆔 Telegram ID: 123456789
📱 Ник: @username
```

Кнопки: **✅ Одобрить** / **❌ Отклонить**


**Одобрить** — сотрудник получает доступ к боту и может вносить смены.  
**Отклонить** — сотрудник получит отказ и инструкцию обратиться к администратору.

После нажатия кнопка исчезает, в тексте появляется пометка кто и когда обработал заявку.

> **Важно:** не нажимай кнопку дважды — это вызовет ошибку. Одного нажатия достаточно.

## Одобрение смен Официанта

### Смена с фото

Если Официант отправил смену вместе с фото, администратору приходит медиа-группа с фотографиями и сообщение:

```
👤 @username — Официант
📅 13.03.26
⏱ 10:00–20:00 → Часы смены = 10 ч
📎 Приложено фото: 3

✅ Сколько фото засчитать как доп. часы?
```

Кнопки: **0** / **1** / **2** / **3** (числа от 0 до количества фото)

Нажми нужное число. Каждое засчитанное фото = **0.5 ч** доп. часов для сотрудника.

После нажатия Официанту уйдёт уведомление с итогом.

### Смена без фото

Если Официант внёс смену без фото — она записывается автоматически. Тебе всё равно приходит информационное уведомление:

```
📋 Официант внёс смену

👤 @username
📅 13.03.26
⏱ 10:00–20:00 → Часы смены = 10 ч
```

Никаких действий не требуется.

## Одобрение карт лояльности

После того как Официант отправил фото карт, тебе придёт уведомление:

```
🎴 Карты лояльности от @username
📅 13.03.26
📸 Фото: 3 шт
```

Кнопки: **0** / **1** / **2** / **3**

Нажми сколько карт засчитать. Каждая карта = **0.5 ч** доп. часов для сотрудника.

---

## Одобрение наполняемости чеков

После того как Официант отправил фото чеков, тебе придёт уведомление:

```
💳 Наполняемость чеков от @username
📅 13.03.26
📸 Фото: 2 шт
```

Кнопки: **1** / **2** *(кнопка «0» недоступна — нельзя одобрить 0 чеков)*

Нажми нужное число. После этого тебе придёт сводка по пулу:

```
✅ Наполняемость записана
Дата: 13.03.26
Добавлено: 2 чека
Всего за первую половину: 45 чеков (67500 р)
```

Чеки попадают в общий пул всего отдела.

## /rates — ставки отдела

Напиши `/rates`. Бот покажет ставки всех сотрудников твоего отдела:

```
💰 Ставки отдела Зал:

• Иванов Иван (Официант): 350 р/ч
  📅 С 1 июня: 400 р/ч
• Петров Пётр (Раннер): 200/300 р/ч (выходные)
• Сидорова Анна (Хостесс): не установлена
```

Пояснения:
- `350 р/ч` — одна ставка для большинства позиций
- `200/300 р/ч (выходные)` — базовая / повышенная ставка для Раннера (выходные дни), Бармена, Барбэка (тусовочные часы)
- `📅 С 1 июня` — запланированное изменение ставки
- `не установлена` — ставка ещё не назначена, сотрудник не видит заработок

## /set_rate — изменить ставку сотрудника

### Шаг 1 — выбор позиции

Напиши `/set_rate`. Бот покажет список позиций твоего отдела:

```
Выберите позицию (Зал):
```

Кнопки: **Менеджер** / **Официант** / **Раннер** / **Хостесс** / ... / **❌ Отмена**

> *Администратор Зала также видит позиции МОП: Клининг и Котломой.*

---

### Шаг 2 — выбор сотрудника

После выбора позиции бот покажет список сотрудников:

```
Выберите сотрудника (Официант):
```

Кнопки с именами сотрудников / **❌ Отмена**

---

### Шаг 3 — период применения

```
Когда применить изменение ставки?
```

Кнопки: **📅 С текущего месяца** / **📆 Со следующего месяца** / **❌ Отмена**

---

### Шаг 4 — ввод ставки

**Для большинства позиций** (Официант, Хостесс, Менеджер, вся Кухня, МОП):

```
Введите ставку (р/ч).
Изменение вступит в силу с 1-го числа текущего месяца.

Формат: целое или дробное число (например: 350 или 450.5)
```

**Для позиций с двойной ставкой** (Раннер, Бармен, Барбэк) — два шага подряд:

Сначала базовая:
```
Введите базовую ставку (обычные дни, р/ч).
```

Потом повышенная:
```
Введите повышенную ставку (выходные, р/ч).
```

*(Для Бармена/Барбэка вместо «выходные» — «тусовочные»)*

---

### Итог

```
✅ Ставка установлена с текущего месяца:
350 р/ч
```

или для двойной ставки:

```
✅ Ставка установлена со следующего месяца:
Базовая: 350 р/ч
Повышенная (выходные): 500 р/ч
```

> **Совет:** если ставка ещё не была установлена новому сотруднику — он видит свои часы, но не видит сумму заработка. Установи ставку как можно скорее после одобрения заявки.

## /message_dept — рассылка по отделу

### Администратор Зала

Напиши `/message_dept`. Бот спросит:

```
Выберите отдел для рассылки:
```

Кнопки: **Зал** / **МОП** / **❌ Отмена**

После выбора введи текст сообщения. Бот разошлёт его всем сотрудникам выбранного отдела.

Сотрудники получат:

```
📢 Сообщение от администратора Зала

[твой текст]
```

Тебе придёт подтверждение: `✅ Сообщение отправлено 12 сотрудникам отдела Зал`


### Администратор Бара / Кухни

Выбора отдела нет — напиши `/message_dept`, сразу введи текст. Рассылка уйдёт по твоему отделу.

## Своя смена и свои часы

Администратор — тоже сотрудник. Команды `/shift`, `/hours_first`, `/hours_second`, `/hours_last` работают для твоих собственных смен точно так же, как для обычного сотрудника.

Обратись к инструкции своей позиции:
- Менеджер или Хостесс → [инструкция Зала](зал.md)
- Бармен или Барбэк → [инструкция Бара](бар.md)
- Позиция Кухни → [инструкция Кухни](кухня.md)

## Различия между типами администраторов

| | admin_hall | admin_bar | admin_kitchen |
|---|---|---|---|
| Отдел | Зал + МОП | Бар | Кухня |
| Заявки | Зал + МОП | Бар | Кухня |
| Уведомления о сменах | Зал + МОП | Бар | Кухня |
| /rates | Зал + МОП | Бар | Кухня |
| /set_rate | Зал + МОП | Бар | Кухня |
| /message_dept | Зал или МОП (выбор) | Бар | Кухня |

> admin_hall управляет двумя подразделениями: Залом и МОП. МОП не имеет своего отдельного администратора.

## 🛟 Советы администратору

- **Не нажимай кнопки одобрения дважды** — это вызовет ошибку
- **Новый сотрудник одобрен, но не видит заработок** → нужно установить ставку через `/set_rate`
- **Не можешь найти сотрудника в /set_rate** → он ещё не одобрен или уволен
- **Бот завис** → `/cancel` → начни заново
- **Технические проблемы** → `/contact_dev`
