# 📋 Инструкция — Суперадмин

> Это руководство для **суперадминов** — управляющих с полным доступом к системе.  
> Суперадмин видит всё и управляет всем: сотрудниками, администраторами, ставками и переключением месяца.

---

## Доступные команды

| Команда | Что делает |
|---|---|
| `/promote` | Повысить сотрудника до администратора отдела |
| `/demote` | Понизить администратора обратно до сотрудника |
| `/dismiss` | Уволить сотрудника или администратора |
| `/set_rate` | Изменить ставку любому сотруднику |
| `/rates_all` | Ставки всех сотрудников по всем отделам |
| `/message_dept` | Рассылка по конкретному отделу |
| `/message_all` | Рассылка всем сотрудникам сразу |
| `/switch_month` | Переключить на новый месяц |
| `/restore_user` | Восстановить сотрудника в таблице вручную |
| `/schedule` | График работы целиком (PDF) |
| `/sheet` | Ссылка на таблицу с графиком |
| `/contact_dev` | Написать разработчику |

> **Важно:** суперадмин **не вносит смены** и **не видит свои часы** — команды `/shift`, `/hours_*` ему недоступны. Суперадмин — управленческая роль.

## /promote — повысить сотрудника

Повышение сотрудника даёт ему права администратора своего отдела.

### Шаг 1 — выбор отдела

Напиши `/promote`. Бот ответит:

```
⬆️ Повышение сотрудника

Выберите подразделение:
```

Кнопки: **Зал** / **Бар** / **Кухня** / **МОП** / **❌ Отмена**

> *Повышение в МОП даёт роль admin_hall — администратора Зала.*

---

### Шаг 2 — выбор позиции

```
Отдел: Зал

Выберите позицию:
```

Кнопки с позициями отдела / **❌ Отмена**

---

### Шаг 3 — выбор сотрудника

Показываются только обычные сотрудники (не администраторы):

```
Отдел: Зал | Позиция: Официант

Выберите сотрудника:
```

---

### Шаг 4 — подтверждение

```
⬆️ Повысить Иванов Иван (Официант, Зал) до администратора отдела?

После повышения сотрудник получит расширенные права управления отделом.
Это действие можно отменить командой /demote.
```

Кнопки: **✅ Повысить** / **❌ Отмена**

После подтверждения тебе придёт:

```
✅ Иванов Иван повышен до администратора Зал.
Ожидаем ввода email для предоставления доступа к таблице.
```

Сотрудник получит уведомление и запрос на ввод Gmail-адреса для доступа к Google Sheets.

### Шаг 5 — email сотрудника

Сотрудник вводит свой Gmail. Тебе придёт:

```
📧 Новый администратор @username ввёл email для доступа к таблице:
example@gmail.com

Добавьте его как редактора в Google Sheets.
```

**После этого вручную добавь этот адрес как редактора в Google Sheets** — бот это не делает автоматически.

## /demote — понизить администратора

Возвращает администратора к роли обычного сотрудника.

### Шаги

1. Напиши `/demote`
2. Выбери отдел
3. Выбери администратора из списка
4. Подтверди:

```
📉 Понизить Иванов Иван (Официант, Зал)?

Сотрудник потеряет права администратора и вернётся к обычным функциям сотрудника своей позиции.
Это действие можно отменить командой /promote.
```

Кнопки: **✅ Да, понизить** / **❌ Отмена**

После подтверждения сотрудник получит уведомление о понижении. Другим суперадминам придёт:

```
📉 Иванов Иван понижен с должности администратора Зал до сотрудника.
📋 Не забудьте удалить его email из редакторов Google Sheets.
```

> **Не забудь** вручную удалить доступ к Google Sheets — бот это не делает.

## /dismiss — уволить сотрудника

Полное удаление сотрудника из системы. **Действие необратимо.**

### Шаг 1 — тип

```
Выберите, кого нужно уволить:
```

Кнопки: **👤 Сотрудник** / **🔑 Администратор отдела** / **❌ Отмена**

### Шаг 2 — отдел и сотрудник

Выбери отдел → выбери конкретного человека из списка.

### Шаг 3 — подтверждение

**Если это обычный сотрудник:**

```
⚠️ Уволить Иванов Иван (Официант, Зал)?

Это действие нельзя отменить.
```

Кнопки: **✅ Да, уволить** / **❌ Отмена**

**Если это администратор — предлагается выбор:**

```
⚠️ Иванов Иван является администратором отдела Зал.

Что вы хотите сделать?
```

Кнопки: **📉 Только понизить до сотрудника** / **🔥 Уволить из системы** / **❌ Отмена**

### Что происходит при увольнении

- Сотрудник получает уведомление об отзыве доступа
- Все его команды в боте сбрасываются
- Запись удаляется из базы данных
- В таблице строка помечается красным цветом (история сохраняется)
- При следующем переключении месяца красная строка не переносится

> **Не забудь** вручную удалить доступ к Google Sheets, если сотрудник был администратором.

## /set_rate — установить ставку

Суперадмин может устанавливать ставки сотрудникам **любого отдела**.

### Шаги

1. Напиши `/set_rate`
2. Выбери отдел: **Зал** / **Бар** / **Кухня** / **МОП**
3. Выбери позицию
4. Выбери сотрудника
5. Выбери период: **📅 С текущего месяца** / **📆 Со следующего месяца**
6. Введи ставку

Для большинства позиций — одно число. Для Раннера, Бармена, Барбэка — два числа подряд: сначала базовая, потом повышенная.

Итог:

```
✅ Ставка установлена со следующего месяца:
Базовая: 350 р/ч
Повышенная (тусовочные): 500 р/ч
```

Напиши `/message_dept`. Выбери отдел:

```
Выберите отдел для рассылки:
```

Тебе придёт подтверждение с числом получателей.

## /switch_month — переключить месяц

Эта команда создаёт новый лист в таблице для следующего месяца и переносит туда активных сотрудников.

> **Обычно переключение происходит автоматически** 1-го числа каждого месяца в 18:00. Ручное переключение нужно только в исключительных случаях.

### Как использовать

Напиши `/switch_month`. Бот запросит подтверждение:

```
⚠️ Переключение месяца

Будет создан новый лист Июнь 2026.
Все активные сотрудники будут перенесены.
Уволенные сотрудники (красные строки) — удалены.

Продолжить?
```

Кнопки: **✅ Да, переключить** / **❌ Отмена**

После нажатия «Да»:

```
⏳ Переключаю месяц, подождите...
```

По завершении:

```
✅ Готово!

Старый лист: Май 2026
Новый лист: Июнь 2026
Перенесено сотрудников: 45
Удалено уволенных: 3
```

### Что происходит при переключении

- Создаётся новый лист с названием следующего месяца
- Все активные сотрудники переносятся с обнулёнными данными смен
- Красные строки (уволенные) не переносятся
- Ставки сотрудников сохраняются; запланированные изменения ставок вступают в силу
- Фантомный сотрудник «Наполняемость чека» переносится с обнулёнными данными

> **Внимание:** действие необратимо. Убедись, что все смены за текущий месяц внесены корректно перед переключением.

## /restore_user — восстановить сотрудника

Если строка сотрудника в таблице была случайно удалена вручную, он не сможет вносить смены. Эта команда восстанавливает его запись.

### Как использовать

Напиши `/restore_user`. Бот попросит:

```
Введите Telegram ID сотрудника для восстановления:
```

Введи числовой Telegram ID сотрудника.

**Если всё хорошо:**

```
✅ Сотрудник Иванов Иван (123456789) восстановлен в текущем месячном листе
```

**Если не найден в системе:**

```
❌ Пользователь 123456789 не найден в Техлисте
```

> Telegram ID можно узнать через [@userinfobot](https://t.me/userinfobot) или попросить сотрудника отправить его.

## 🛟 Советы суперадмину

- **После /promote** → вручную добавить email нового администратора в Google Sheets
- **После /demote или /dismiss администратора** → вручную удалить его email из Google Sheets
- **Перед /switch_month** → убедиться, что все смены за месяц внесены
- **Сотрудник не может внести смену** → возможно его строка в таблице удалена → использовать `/restore_user`
- **Бот завис** → `/cancel` → начни заново
- **Технические проблемы** → `/contact_dev`
